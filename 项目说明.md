# a-pipelinw-for-kaggle 项目说明

这个项目是一个用于 Kaggle 风格文本评分预测任务的基础机器学习流水线，当前实现面向 Amazon 电影评论星级预测。它的目标不是追求最复杂的模型，而是提供一套容易理解、容易复现、容易继续扩展的训练与提交流程。

项目目前采用的是传统机器学习方案，而不是深度学习或 boosting 模型。整体设计思路是把不同类型的输入信号拆开处理，再交给一个统一的 sklearn `Pipeline` 完成训练和预测。

## 代码结构

- `src/common.py`
  放共享逻辑，包括字段推断、特征工程、预处理器构建、评估函数和预测后处理。

- `src/train.py`
  训练入口。负责读取训练数据、筛选有标签样本、构建特征、交叉验证、留出验证、训练最终模型并保存指标。

- `src/predict.py`
  推理入口。负责读取测试集顺序、对无标签样本生成特征、加载训练好的模型并输出提交文件。

## 数据流说明

这个比赛的数据组织方式比较特殊：

- `train.csv` 中同时包含有标签样本和无标签样本
- 有标签样本的 `Score` 不为空，可以用于训练
- 无标签样本的 `Score` 为空，需要被预测
- `test.csv` 并不是完整特征表，主要作用是提供需要提交的 `Id` 以及提交顺序

因此，这个项目在训练和预测时的处理方式如下：

1. 训练阶段只使用 `train.csv` 中 `Score` 非空的行。
2. 预测阶段从 `train.csv` 中取出 `Score` 为空的行。
3. 再按照 `test.csv` 中的 `Id` 顺序重排这些无标签样本。
4. 最终输出符合 Kaggle 提交格式的 `submission.csv`。

## 特征工程思路

当前流水线把特征分成三类：

- 文本特征
  例如 `Summary`、`Text`。这些列会先合并成 `merged_text`，再用 `TF-IDF` 转成稀疏向量。

- 类别特征
  例如 `ProductId`、`UserId`。这些列使用 `OneHotEncoder` 编码，并忽略预测时未见过的新类别。

- 数值特征
  例如帮助度计数、时间戳等。数值列会先填补缺失值，再进行缩放。

除了原始列之外，项目还会补充一些简单但常见的人工特征，例如：

- 文本字符数
- 文本词数
- 感叹号数量
- 问号数量
- 大写字母占比
- 数字字符占比

这些特征的作用，是为模型提供一些 `TF-IDF` 之外的低维统计信息。

## 模型与评估

当前模型使用的是 `Ridge` 回归。原因是它对高维稀疏文本特征比较稳定，训练速度也快，适合作为一个可复现的基线模型。

评估部分包含两种结果：

- 交叉验证 `RMSE`
  用来观察模型在多折划分下的整体稳定性。

- 留出验证 `RMSE`
  用来模拟模型在未见样本上的单次泛化表现。

因为目标值是星级分数，模型预测出的连续值在最终提交前会进行：

- 四舍五入
- 裁剪到训练集标签范围内

这样可以保证输出结果是合法的整数评分。

## 适合继续扩展的方向

- 增加更多时间相关特征
- 增加帮助度衍生特征
- 尝试更强的非 boosting 线性模型或核方法
- 调整 `TF-IDF` 参数
- 做更细的交叉验证和超参数搜索
- 补充实验记录与结果对比

## 适用场景

这个项目适合以下几类用途：

- 作为 Kaggle 比赛的基础提交模板
- 作为课程作业中的传统机器学习 baseline
- 作为后续特征工程实验的起点
- 作为理解 sklearn 训练/预测流水线的示例项目
